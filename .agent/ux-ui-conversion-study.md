## 1. Fundamentos de UX para Conversão

### 1.1 O ROI do UX Design

O retorno financeiro do investimento em UX é um dos mais altos em tecnologia:

| Métrica | Dado | Fonte |
|---------|------|-------|
| ROI médio por $1 investido em UX | **$100** de retorno | Forrester Research / Nielsen Norman Group |
| Aumento de conversão com UX positiva | Até **400%** | UX Planet / NN/g Research |
| Aumento de conversão com 10% investimento em UX | **83%** de lift | UX Planet |
| Aumento de ROI com design centrado no usuário | Até **228%** | Nielsen Norman Group |

> **Insight-chave:** UX não é custo — é investimento. Cada real aplicado em pesquisa de usuário, teste de usabilidade e design centrado no usuário retorna exponencialmente em conversões e receita.

### 1.2 As 10 Heurísticas de Usabilidade de Jakob Nielsen

As heurísticas de Nielsen (Nielsen Norman Group) são o alicerce de qualquer projeto de UI profissional:

1. **Visibilidade do status do sistema** — O sistema deve manter o usuário informado sobre o que está acontecendo, em tempo real
2. **Correspondência com o mundo real** — Linguagem familiar, conceitos conhecidos, convenções do mundo real
3. **Controle e liberdade do usuário** — "Saída de emergência" sempre disponível (desfazer, voltar)
4. **Consistência e padrões** — Mesmas ações = mesmos resultados em todo o sistema
5. **Prevenção de erros** — Design que elimina condições propensas a erro antes que aconteçam
6. **Reconhecer em vez de lembrar** — Minimizar carga de memória; tornar opções visíveis
7. **Flexibilidade e eficiência** — Atalhos para usuários experientes, sem poluir para novatos
8. **Design estético e minimalista** — Cada elemento extra compete com os elementos relevantes
9. **Ajudar usuários a reconhecer e recuperar erros** — Mensagens de erro em linguagem simples, com solução sugerida
10. **Ajuda e documentação** — Fácil de buscar, focada na tarefa, concisa

### 1.3 Princípios Fundamentais de UX para Conversão

Baseado em pesquisas do CXL Institute, Baymard Institute, e NN/g:

- **Clareza > Cleverness** — O visitante deve entender o que a empresa faz em menos de 5 segundos
- **Arquitetura de informação intuitiva** — Estrutura lógica, hierarquia clara, navegação óbvia
- **Redução de fricção** — Eliminar etapas desnecessárias, simplificar processos, minimizar decisões
- **Personalização** — Experiências relevantes baseadas em comportamento, localização e contexto
- **Feedback imediato** — Cada ação do usuário deve ter uma resposta visual instantânea (< 100ms)
- **Mobile-first** — Mais de 50% do tráfego web é mobile; design para menor tela primeiro

## 2. Psicologia da Persuasão no Web Design

### 2.1 Os 7 Princípios de Influência de Robert Cialdini

O trabalho de Dr. Robert Cialdini é fundamental para entender como design persuade:

#### Reciprocidade
> "Dê algo de valor antes de pedir algo em troca"

**Aplicação prática:**
- Oferecer conteúdo gratuito (e-books, guias, ferramentas) antes de pedir dados
- Calculadoras de custo ou orçamento gratuitas
- No contexto Macaster: Oferecer um **orçamento instantâneo** antes de solicitar contato

#### Comprometimento e Consistência
> "Pequenos compromissos levam a grandes ações"

**Aplicação prática:**
- Solicitar micro-ações antes da macro-conversão (ex: "veja os preços" → "calcule seu orçamento" → "fale com consultor")
- Usar formulários multi-step que começam com perguntas simples
- No contexto Macaster: Botão "Cotar Materiais" como primeiro passo de baixo compromisso

#### Prova Social
> "Se outros confiam, devo confiar também"

**Aplicação prática:**
- Depoimentos com foto e nome real (+34% conversão)
- Logos de parceiros/clientes reconhecidos
- Contadores: "X obras entregues", "Y clientes satisfeitos"
- No contexto Macaster: Logo cloud de parceiros (NVIDIA, etc.) já implementado ✅

#### Autoridade
> "Especialistas sabem mais que eu"

**Aplicação prática:**
- Certificações e selos de qualidade
- Tempo de mercado ("+20 anos de experiência")
- Depoimentos de figuras de autoridade no setor

#### Afeição (Liking)
> "Compramos de quem gostamos"

**Aplicação prática:**
- Visual moderno e atraente que reflete profissionalismo
- Storytelling sobre a empresa, equipe e valores
- Tom de comunicação acessível e humano

#### Escassez
> "Se é limitado, deve ser valioso"

**Aplicação prática:**
- "Vagas limitadas para entrega imediata"
- Countdowns para promoções ("preço especial até [data]")
- Indicadores de estoque ("últimas X unidades disponíveis")

#### Unidade
> "Somos parte do mesmo grupo"

**Aplicação prática:**
- Linguagem inclusiva ("nós", "juntos", "nosso parceiro")
- Identidade regional ou setorial compartilhada
- Comunidade: "Junte-se a X construtoras que já confiam em nós"

### 2.2 Modelo Comportamental de Fogg (B.J. Fogg, Stanford)

O modelo Fogg define que **Comportamento = Motivação × Habilidade × Gatilho** (B = MAT).

```
    Alta  │      ✅ Ação acontece
          │      (Motivação alta + Fácil + Gatilho)
Motivação │
          │  ❌ Sem ação
          │  (Motivação baixa + Difícil)
    Baixa └──────────────────────────────
          Difícil                    Fácil
                  Habilidade (Facilidade)
```

**Aplicação no web design:**

| Componente | O que é | Como otimizar no site |
|------------|---------|----------------------|
| **Motivação** | Desejo de agir | Headlines que comunicam valor claro, imagens emocionais, urgência |
| **Habilidade** | Facilidade de agir | Formulários curtos, botões óbvios, zero fricção, CTA claro |
| **Gatilho** | O empurrão para agir | CTAs visíveis no momento certo, posição estratégica, cor contrastante |

> **Insight-chave:** Se a motivação é alta mas a ação é difícil, a conversão não acontece. Se a ação é fácil mas não há motivação, também não. O design deve equilibrar ambos e posicionar gatilhos (CTAs) no momento de pico motivacional.

## 3. Hierarquia Visual e Padrões de Leitura

### 3.1 Eye-Tracking: Como os olhos escaneiam uma página

Estudos de eye-tracking (Nielsen Norman Group, Tobii, etc.) revelaram padrões previsíveis:

#### Padrão F (F-Pattern)
```
████████████████████████████
████████████████████
████████████
████
████
████
████
```
- **Onde:** Páginas com texto denso (blogs, artigos, páginas de produto)
- **Comportamento:** Leitura horizontal no topo → segunda passada mais curta → escaneamento vertical pela esquerda
- **Implicação:** Informações críticas devem estar no topo-esquerda

#### Padrão Z (Z-Pattern)
```
1 ━━━━━━━━━━━━━ 2
                ╲
                 ╲
                  ╲
3 ━━━━━━━━━━━━━ 4
```
- **Onde:** Landing pages minimalistas, páginas com pouco texto
- **Comportamento:** Topo-esquerda → topo-direita → diagonal até embaixo-esquerda → embaixo-direita
- **Implicação:** Ideal para landing pages de conversão — CTA no ponto 4 (terminal area)

#### Diagrama de Gutenberg
```
┌─────────────┬─────────────┐
│ 🔴 Primary  │ ⚪ Strong   │
│  Optical    │  Fallow     │
├─────────────┼─────────────┤
│ ⚪ Weak     │ 🔴 Terminal │
│  Fallow     │  Area       │
└─────────────┴─────────────┘
```
- **Primary Optical Area** (topo-esquerda): Logo, título principal → onde os olhos começam
- **Terminal Area** (embaixo-direita): CTA → onde os olhos terminam naturalmente
- **Strong Fallow** (topo-direita): Navegação, busca
- **Weak Fallow** (embaixo-esquerda): Área menos vista — não colocar info crítica aqui

### 3.2 Princípios de Hierarquia Visual

1. **Tamanho e peso** — Elementos maiores e mais pesados (bold) são vistos primeiro
2. **Cor e contraste** — Alto contraste atrai o olho; cores vibrantes sobre fundos neutros
3. **Posição** — Topo > fundo; esquerda > direita (em culturas ocidentais)
4. **Espaçamento** — Mais whitespace ao redor = mais importância percebida
5. **Proximidade** — Elementos próximos são percebidos como relacionados (Gestalt)
6. **Repetição** — Padrões visuais consistentes criam ritmo e previsibilidade

### 3.3 Aplicação Prática para Macaster

```
┌─────────────────────────────────────────────┐
│ [Logo] ──── Nav ──── [CTA: Cotar Agora] 🔴  │  ← Z-ponto 1→2
│                                              │
│       Título Principal (H1)                  │  ← F-topo
│       Subtítulo com proposta de valor        │
│       [CTA PRIMÁRIO] [CTA secundário]        │  ← Z-ponto 3→4
│                                              │
│       Logos de Parceiros (social proof)       │
└─────────────────────────────────────────────┘
```

## 4. Design do Hero Section e "Above the Fold"

### 4.1 O que é "Above the Fold"

Termo herdado de jornais — a parte visível antes do scroll. Estudos mostram:

- **57% do tempo de visualização** é gasto above the fold (Chartbeat / NN/g)
- Visitantes formam opinião sobre o site em **50 milissegundos** (pesquisa do Ontario Tech University, 2006)
- A decisão de ficar ou sair acontece em **3-5 segundos**

### 4.2 Elementos Essenciais do Hero

| Elemento | Função | Best Practice |
|----------|--------|---------------|
| **Headline (H1)** | Comunicar proposta de valor | Curta (5-10 palavras), benefício claro, sem jargão |
| **Subheadline** | Expandir e qualificar | 1-2 frases, como o produto/serviço resolve o problema |
| **CTA Primário** | Converter | Verbo de ação, cor contrastante, acima do fold |
| **CTA Secundário** | Alternativa de baixo compromisso | "Saiba mais", "Veja como funciona" |
| **Visual** | Criar emoção e contexto | Imagem/vídeo relevante, alta qualidade |
| **Trust signals** | Reduzir ansiedade | Logos, selos, números de credibilidade |

### 4.3 Erros Comuns no Hero

❌ Headline genérica ("Bem-vindo ao nosso site")  
❌ Imagem stock genérica sem conexão com o produto  
❌ CTA abaixo do fold  
❌ Múltiplos CTAs competindo pela atenção  
❌ Texto demais — hero sobrecarregado  
❌ Falta de proposta de valor clara  

### 4.4 Estudo de Caso: Hero Sections de Alta Conversão

**Stripe** — Hero minimalista, headline orientada a benefício ("Pagamentos integrados para a internet"), CTA claro, animação de código ao vivo, background clean

**Linear** — Headline curta e direta, subtítulo que qualifica, dois CTAs (primário + secundário), visual do produto, dark mode elegante

**Vercel** — "Develop. Preview. Ship." — Headline de 3 palavras, poder do minimalismo, gradient text como destaque visual

## 5. Calls-to-Action (CTAs) que Convertem

### 5.1 Anatomia de um CTA de Alta Performance

```
┌─────────────────────────────────┐
│  ▶  Cotar Materiais Agora       │  ← Verbo de ação + contexto + urgência
│                                  │
│  Cor contrastante (#F39200)      │  ← Destaca do background
│  Tamanho mínimo 48×48px         │  ← Touch-friendly
│  Padding generoso (16px+)       │  ← Área de clique ampla
│  Whitespace ao redor            │  ← Respira, chama atenção
└─────────────────────────────────┘
```

### 5.2 Dados de A/B Testing sobre CTAs

| Teste | Resultado | Fonte |
|-------|-----------|-------|
| CTA verde → vermelho | **+21%** conversões | HubSpot |
| CTAs animados vs. estáticos | **+15-40%** click-through | CleverQuokka / Educational Voice |
| CTA personalizado vs. genérico | **+202%** conversões | HubSpot |
| "Submit" → texto específico de ação | **+3-10%** conversões | Unbounce |
| Botão bounce animado em "Add to Cart" | **+23%** compras | Estudo e-commerce |

### 5.3 Regras de Ouro para CTAs

1. **Verbo de ação** — "Cotar", "Solicitar", "Ver preços" (não "Enviar" ou "Clique aqui")
2. **Benefício implícito** — "Receba seu orçamento" > "Enviar formulário"
3. **Cor com contraste extremo** — O botão deve ser a coisa mais visível da seção
4. **Um CTA primário por seção** — Múltiplos CTAs de igual peso = paralisia decisória
5. **Hierarquia visual** — Primário (filled/sólido) > Secundário (outline/ghost)
6. **Tamanho mobile-friendly** — Mínimo 44×44px (Apple HIG) ou 48×48px (Material Design)
7. **Repetição ao longo da página** — CTA above the fold, no meio e no final
8. **Micro-copy de suporte** — "Sem compromisso" ou "Resposta em 24h" abaixo do botão

### 5.4 Posicionamento

- **Above the fold** — Visibilidade imediata
- **Após proposta de valor** — O usuário precisa entender o benefício antes de agir
- **Final de cada seção** — Capturar quem foi convencido naquela seção
- **Sticky/fixed** — Para mobile, CTA flutuante no bottom da tela

#### Tier 3 — Contextual
- **Cases de sucesso** — Detalhamento de projetos específicos
- **Conteúdo UGC** — Fotos reais de obras (user-generated content)
- **Press mentions** — "Visto em [mídia]"

## 7. Tipografia e Legibilidade

### 7.1 Regras Fundamentais

| Propriedade | Valor Recomendado | Por quê |
|-------------|-------------------|---------|
| **Body text** | 16-18px mínimo | Legibilidade universal (Smashing Magazine) |
| **Line-height** | 1.5-1.6 | Máxima legibilidade para texto corrido |
| **Heading line-height** | 1.1-1.3 | Headings grandes precisam de menos espaço |
| **Measure (largura)** | 50-75 caracteres | Além disso, o olho perde a linha (Typographic Style) |
| **Famílias de fonte** | Máximo 2 | Consistência e coerência visual |
| **CTA text** | 18-22px | Clareza e impacto |
| **Contraste texto/fundo** | 4.5:1 mínimo (WCAG AA) | Acessibilidade e legibilidade |

### 7.2 Hierarquia Tipográfica

```
H1 — Título Principal
    Tamanho: 40-72px | Peso: 700-800 | Tracking: -0.02 a -0.03em
    Uso: Uma vez por página. Statement principal.

H2 — Título de Seção
    Tamanho: 28-40px | Peso: 600-700 | Tracking: -0.01 a -0.02em
    Uso: Dividir grandes seções de conteúdo

H3 — Subtítulo/Card Title
    Tamanho: 20-28px | Peso: 500-600
    Uso: Títulos de cards, sub-seções

Body — Texto corrido
    Tamanho: 16-18px | Peso: 400 | Line-height: 1.5-1.6
    Uso: Parágrafos, descrições

Caption — Texto auxiliar
    Tamanho: 12-14px | Peso: 400 | Color: 60% opacity
    Uso: Labels, metadata, notas de rodapé
```

### 7.3 Fontes Recomendadas para Conversão

| Categoria | Fontes | Personalidade |
|-----------|--------|---------------|
| **Sans-serif moderna** | Inter, SF Pro, Outfit, Geist | Clean, tech, premium |
| **Sans-serif humanista** | Plus Jakarta Sans, Albert Sans | Amigável, acessível |
| **Serifa moderna** | Playfair Display, Fraunces | Elegante, sofisticada |
| **Monospace** | JetBrains Mono, Geist Mono | Técnica, precisa |

### 7.4 Anti-Patterns Tipográficos

❌ Fontes decorativas em corpo de texto  
❌ Texto menor que 16px em body  
❌ ALL CAPS em parágrafos (reduz velocidade de leitura em 17%, Nielsen 1998)  
❌ Low contrast (texto cinza claro em fundo branco)  
❌ Fontes extremamente finas (weight 100-200) em body  
❌ Mais de 3 famílias de fonte diferentes

## 8. Psicologia das Cores

### 8.1 Cores e Emoções

| Cor | Emoção / Associação | Uso em Web | Exemplo de marca |
|-----|---------------------|------------|-----------------|
| 🔵 **Azul** | Confiança, segurança, profissionalismo | Bancos, saúde, tech | Facebook, LinkedIn, Samsung |
| 🔴 **Vermelho** | Urgência, paixão, energia, ação | CTAs, promoções, alertas | YouTube, Netflix, Coca-Cola |
| 🟢 **Verde** | Natureza, crescimento, sucesso, saúde | Eco, finanças, wellness | Spotify, Starbucks, WhatsApp |
| 🟡 **Amarelo** | Otimismo, atenção, calor | Alertas, destaques (com moderação) | IKEA, McDonald's, Snapchat |
| 🟠 **Laranja** | Energia, entusiasmo, CTA friendly | CTAs, destaques de preço | Amazon (CTA), HubSpot, Macaster |
| 🟣 **Roxo** | Luxo, criatividade, sofisticação | Premium, beauty, inovação | Twitch, Cadbury, Nubank |
| ⚫ **Preto** | Elegância, poder, premium | Luxury, fashion, tech | Apple, Nike, Chanel |
| ⚪ **Branco** | Limpeza, simplicidade, espaço | Minimalismo, saúde | Apple, Google |

### 8.2 Dados de Pesquisa sobre Cores

- **90%** das avaliações de primeira impressão de um produto podem ser baseadas apenas na cor (Singh, 2006)
- **62-90%** da primeira impressão sobre um site é determinada pela cor (Entrepreneur)
- Cores consistentes aumentam **reconhecimento de marca em 80%** (University of Loyola)
- Mudança de CTA de verde → vermelho: **+21% conversões** (HubSpot A/B test)
- CTAs vermelhos e laranjas geraram **32-40% mais cliques** que outras cores (Striven)

### 8.3 Paleta de Cores para Conversão do Macaster

```
Primária: #F39200 (Laranja Macaster) — Energia, ação, CTAs
    ├── Hover: lighten 10% → mais vibrante
    └── Active: darken 5% → feedback tátil

Secundária: #FFFFFF (Branco) — Limpeza, contraste, premium
    ├── Text: rgba(255,255,255,0.9) → legibilidade máxima
    └── Muted: rgba(255,255,255,0.6) → texto secundário

Background: #000000 → #0A0A0A (Preto) — Premium, elegante
    ├── Surface: rgba(255,255,255,0.05) → cards glass
    └── Border: rgba(255,255,255,0.10) → separadores sutis

Accent: Evitar azul forte (mesma cor que links padrão do browser)
Sucesso: #22C55E (Verde) — Confirmações, ícones de benefício
Erro: #EF4444 (Vermelho) — Validação de formulários
```

### 8.4 Regra do 60-30-10

Padrão da indústria para aplicação proporcional de cores:
- **60%** — Cor dominante (background: preto/escuro)
- **30%** — Cor secundária (texto: branco, surfaces)
- **10%** — Cor de destaque/CTA (laranja Macaster)

## 9. Whitespace e Composição

### 9.1 O Poder do Espaço Negativo

Whitespace (espaço negativo) é uma das ferramentas mais subestimadas no web design:

| Dado | Estatística | Fonte |
|------|-------------|-------|
| Atenção visual | **+35-45%** mais atenção com whitespace adequado | Seers.co |
| Percepção de marca | Whitespace eleva percepção de **premium** e **sofisticação** | Apple Design |
| Compreensão leitora | **+20%** com espaçamento adequado entre linhas | Wichita State University |
| Foco no CTA | Whitespace isola o CTA, **aumentando cliques** | AttentionInsight |

### 9.2 Tipos de Whitespace

1. **Macro whitespace** — Margens, padding entre seções, espaço entre blocos de conteúdo
2. **Micro whitespace** — Line-height, letter-spacing, padding interno de botões e cards

### 9.3 Diretrizes Práticas

```css
/* Espaçamento entre seções */
section { padding: 80px 0; }     /* Mobile: 48px */

/* Espaçamento entre blocos */
.content-block + .content-block { margin-top: 48px; }

/* Espaçamento interno de cards */
.card { padding: 32px; }

/* CTA breathing room */
.cta-section { padding: 64px 0; }

/* Máxima largura de conteúdo */
.container { max-width: 1200px; }

/* Largura de leitura */
.prose { max-width: 680px; }  /* ~65 caracteres */
```

### 10.2 Os 4 Pilares de uma Micro-Interação (Dan Saffer)

```
1. TRIGGER → O que inicia (clique, hover, scroll, timer)
2. RULES   → O que acontece (animação, mudança de estado)
3. FEEDBACK → O que o usuário vê/sente (visual, sonoro, tátil)
4. LOOPS    → Como termina ou repete (single, loop, estado final)
```

### 10.3 Taxonomia de Animações por Propósito

#### Animações de **Orientação** (guiar o usuário)
- Fade-in on scroll — Elementos aparecem conforme o usuário scrolla
- Stagger reveal — Itens aparecem sequencialmente (direção, timing)
- Progress indicators — Mostram progresso em formulários multi-step
- Scroll-driven navigation — Header que esconde/aparece com scroll

#### Animações de **Feedback** (confirmar ação)
- Button press state — Escala, cor, ou ripple ao clicar
- Form validation — Check verde, shake vermelho
- Loading states — Spinner, skeleton screens, shimmer
- Success/error states — Toasts animados, confetti

#### Animações de **Engajamento** (manter atenção)
- Hero text animations — Typewriter, blur reveal, text fill on scroll
- Counter animations — Números contando até o valor final
- Logo carousel — Continuous scroll suave de logos
- Parallax layers — Profundidade sutil com mousemove

#### Animações de **Deleite** (surpreender)
- Shine sweep — Luz que percorre botões no hover
- Easter eggs — Interações escondidas que recompensam curiosidade
- Ambient animations — Gradientes lentos, partículas sutis
- Cursor effects — Cursor customizado que reage ao contexto

### 10.4 Durações Ideais de Animação

| Tipo | Duração | Easing |
|------|---------|--------|
| **Hover feedback** | 150-250ms | `ease-out` ou `power2.out` |
| **Entrance (fade-in)** | 300-600ms | `ease-out` ou `power3.out` |
| **Page transition** | 200-400ms | `ease-in-out` |
| **Modal open** | 200-350ms | `ease-out` com spring |
| **Scroll reveal** | 400-800ms | `power2.out` |
| **Counter animation** | 1500-2500ms | `power3.out` |
| **Complex sequence** | 800-2000ms total | Timeline com stagger |

> **Regra de ouro:** Animações de feedback (hover, click) devem ser < 300ms. Animações de entrada (scroll reveal) podem ser 400-800ms. Nunca exceda 1s para uma animação individual — o usuário perde interesse.

### 10.5 Anti-Patterns de Animação

❌ Animações que bloqueiam interação (loader que impede scroll)  
❌ Animações que movem CTAs durante hover (botão que "foge" do clique)  
❌ Excesso de animação — mais de 3 elementos animando simultaneamente  
❌ Animações que repetem infinitamente sem propósito  
❌ Animações de entrada lentas (> 1s) que atrasam a mensagem  
❌ Não respeitar `prefers-reduced-motion` (acessibilidade)  
❌ Animações que causam layout shift (CLS ruim para SEO)  

### 10.6 Stack Técnico Recomendado

| Uso | Biblioteca | Justificativa |
|-----|-----------|---------------|
| **Scroll-driven** | GSAP + ScrollTrigger | Performance, controle preciso, scrub |
| **React components** | Motion (Framer Motion) | Declarativo, AnimatePresence, gestures |

| Formulários com UX otimizado | **~2x** completion rate | RapidForms |
| Melhoria no checkout UX | **+35%** conversão em e-commerce | Baymard Institute |

### 12.2 Princípios de Design de Formulários

1. **Mínimo de campos** — Cada campo extra é atrito. Pergunte apenas o essencial.
2. **Layout single-column** — Fluxo vertical claro, sem confusão visual
3. **Labels acima dos campos** — Nunca como placeholder (acessibilidade)
4. **Validação em tempo real** — Feedback instantâneo, inline, não no topo
5. **Botão com ação clara** — "Solicitar Orçamento" (não "Enviar")
6. **Microcopy de suporte** — "Resposta em até 2h" abaixo do botão
7. **Trust signals no form** — "Seus dados estão seguros" + ícone de cadeado
8. **Autofill habilitado** — Usar `autoComplete` nos inputs HTML
9. **Keyboard type correto** — `inputMode="tel"` para telefone, `type="email"` para e-mail

### 12.3 Formulário Multi-Step para Macaster

```
Step 1: "O que você precisa?" (2 campos)
├── Tipo de material (select: Concreto | Madeira | Compensado | Outros)
└── Quantidade estimada (input numérico)

Step 2: "Quando e onde?" (2 campos)
├── Data de entrega desejada (date picker)
└── Cidade/Região (autocomplete)

Step 3: "Como entrar em contato" (2 campos)
├── Seu nome (text)
└── WhatsApp (tel) ← single field, mais usado que email

[Solicitar Orçamento] ← CTA com ação clara
"Sem compromisso • Resposta em até 2h"
```

### 13.2 Checklist de Acessibilidade Essencial

- [ ] **Contraste** — Ratio mínimo 4.5:1 (texto normal) e 3:1 (texto grande)
- [ ] **Navegação por teclado** — Todos elementos interativos acessíveis via Tab
- [ ] **Focus visible** — Indicador de foco claro e visível em todos os elementos
- [ ] **Alt text** — Todas imagens com texto alternativo descritivo
- [ ] **Estrutura semântica** — H1→H2→H3 hierárquico, landmarks ARIA
- [ ] **Labels de formulário** — Cada input com `<label>` associado
- [ ] **Reduced motion** — Respeitar `prefers-reduced-motion`
- [ ] **Tamanho de toque** — Mínimo 44×44px para elementos interativos
- [ ] **Linguagem clara** — Nível de leitura accessível (Flesch-Kincaid)
- [ ] **Gestão de foco** — Focus trap em modais, retorno de foco ao fechar

### 13.3 Implementação CSS para Reduced Motion

```css
@media (prefers-reduced-motion: reduce) {
  *,
  ::before,
  ::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

## 14. Checklists e Diretrizes Práticas

### 14.1 Checklist de Conversão — Landing Page

#### Above the Fold
- [ ] Headline comunica proposta de valor em ≤ 10 palavras
- [ ] Subheadline qualifica e contextualiza o benefício
- [ ] CTA primário visível sem scroll, com cor contrastante
- [ ] CTA secundário presente (baixo compromisso)
- [ ] Visual relevante (produto, resultado, equipe real)
- [ ] Trust signal mínimo (logo cloud ou selo)

#### Hierarquia Visual
- [ ] Z-pattern aplicado (logo → nav → headline → CTA)
- [ ] Um H1 por página (SEO + hierarquia)
- [ ] Heading sizes decrescentes (H1 > H2 > H3)
- [ ] Whitespace adequado entre seções (60-120px)
- [ ] Máximo de leitura de 65-75 caracteres por linha

#### CTAs
- [ ] Verbo de ação específico ("Cotar", não "Enviar")
- [ ] Cor com alto contraste contra o background
- [ ] Tamanho ≥ 44×44px (touch-friendly)
- [ ] Microcopy de suporte ("Sem compromisso", "Grátis")
- [ ] Repetido a cada 2-3 seções na scroll

#### Social Proof
- [ ] Logos de parceiros/clientes
- [ ] Números de credibilidade (counters)
- [ ] Depoimentos com foto + nome + empresa
- [ ] Avaliação/rating visível

#### Performance
- [ ] LCP ≤ 2.5s
- [ ] CLS ≤ 0.1
- [ ] INP ≤ 200ms
- [ ] Imagens otimizadas (WebP, lazy load)
- [ ] Fontes com display:swap

#### Mobile
- [ ] Layout responsivo testado em 375px, 768px, 1440px
- [ ] CTAs easy to tap (48px+)
- [ ] Formulários com keyboard type correto
- [ ] Menu mobile acessível e fácil de fechar

### 14.2 Checklist de Animação — Do's and Don'ts

#### ✅ Do
- [ ] Fade-up on scroll para conteúdo principal (0.4-0.6s)
- [ ] Stagger delay entre cards/items (0.05-0.1s)
- [ ] Hover feedback em < 200ms
- [ ] Counter animation em números de credibilidade
- [ ] Loading skeleton enquanto dados carregam
- [ ] Exit animation em modais/menus (AnimatePresence)
- [ ] Shine effect em botões (CSS pseudo-element)
- [ ] Respeitar prefers-reduced-motion

#### ❌ Don't
- [ ] NÃO mover CTAs durante hover (botão que "foge")
- [ ] NÃO animar mais de 3 elementos simultâneos
- [ ] NÃO usar animações > 1s para elementos individuais
- [ ] NÃO bloquear interação com animações
- [ ] NÃO causar layout shift (anima transform, não width/height)
- [ ] NÃO ignorar mobile (simplificar animações complexas)

### 14.3 Regra de Ouro: O Framework AIDA para Web Design

```
ATTENTION → Hero section com headline poderosa + visual impactante
           (Captura em < 3 segundos)

INTEREST  → Social proof + benefícios detalhados + demonstração
           (Gera curiosidade e credibilidade)

DESIRE    → Depoimentos + cases + comparativos + garantias
           (Constrói vontade de agir)

ACTION    → CTA claro + formulário simplificado + urgência
           (Facilita a conversão)
```
